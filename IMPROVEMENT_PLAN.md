## 二、缺失功能详解

### 2.1 NarrativeMemory (叙事记忆) - ❌ 缺失

**evolver 实现**: `src/gep/narrativeMemory.js`

**功能描述**:
- 记录每次进化的决策过程和结果
- 生成人类可读的进化叙事日志
- 自动修剪旧条目 (MAX_NARRATIVE_ENTRIES = 30, MAX_NARRATIVE_SIZE = 12000)
- 为反思阶段提供上下文

**关键函数**:
```javascript
recordNarrative({ gene, signals, mutation, outcome, blast, capsule })
loadNarrativeSummary(maxChars)  // 加载最近 8 条条目
trimNarrative(content)         // 自动修剪
```

**输出格式** (Markdown):
```markdown
### [2026-03-14 10:23:45] REPAIR - success
- Gene: gene_auto_abc123 | Score: 0.85 | Scope: 3 files, 45 lines
- Signals: [log_error, errsig:TypeError...]
- Why: 修复了 X 问题
- Strategy:
  1. 分析错误日志
  2. 定位问题根源
- Result: 成功修复 TypeError
```

---

### 2.2 Reflection (反思阶段) - ❌ 缺失

**evolver 实现**: `src/gep/reflection.js`

**功能描述**:
- 每 5 个进化周期触发一次战略反思
- 分析近期进化模式的成功/失败趋势
- 生成战略调整建议
- 30 分钟冷却期防止过度反思

**关键配置**:
```javascript
REFLECTION_INTERVAL_CYCLES = 5
REFLECTION_COOLDOWN_MS = 30 * 60 * 1000  // 30 分钟
```

**反思问题**:
1. 是否有持续被忽略的信号？
2. Gene 选择策略是否最优？
3. repair/optimize/innovate 平衡是否需要调整？
4. 是否存在当前 Gene 无法覆盖的能力缺口？
5. 哪个单一战略调整会产生最高影响？

**输出格式** (JSON):
```json
{
  "insights": ["连续 5 次使用 repair gene 但错误持续出现"],
  "strategy_adjustment": "建议尝试 innovate 模式探索新方案",
  "priority_signals": ["errsig:DatabaseConnection"]
}
```

---

### 2.3 LLMReview (LLM 审查) - ❌ 缺失

**evolver 实现**: `src/gep/llmReview.js`

**功能描述**:
- 在 solidify 之前可选的 LLM 代码审查
- 通过 `EVOLVER_LLM_REVIEW=true` 启用
- 30 秒超时保护
- 返回批准/拒绝决定和置信度

**审查标准**:
1. 变更是否解决了声明的信号？
2. 是否引入明显的回归或 bug？
3. blast radius 是否与问题成比例？
4. 是否存在安全或安全隐患？

**输出格式**:
```json
{
  "approved": true,
  "confidence": 0.85,
  "concerns": [],
  "summary": "变更符合预期，无安全问题"
}
```

### 2.5 IssueReporter (自动 Issue 报告) - ❌ 缺失

**evolver 实现**: `src/gep/issueReporter.js`

**功能描述**:
- 检测持续失败模式 (连续 5+ 次失败)
- 自动创建 GitHub Issue 报告问题
- 24 小时冷却期防止刷屏
- 脱敏日志和错误信息

**触发条件**:
- `failure_loop_detected` 信号
- `recurring_error` + `high_failure_ratio` 组合
- 连续失败 >= 5 次

**配置**:
```javascript
EVOLVER_AUTO_ISSUE=true
EVOLVER_ISSUE_REPO=autogame-17/capability-evolver
EVOLVER_ISSUE_COOLDOWN_MS=86400000  // 24h
EVOLVER_ISSUE_MIN_STREAK=5
```

**Issue 格式**:
```markdown
## Environment
- Evolver Version: v1.29.8
- Node.js: v20.10.0
- Platform: linux x64

## Failure Summary
- Consecutive failures: 7
- Failure signals: failure_loop_detected, ban_gene:gene_xxx

## Error Signature
```
TypeError: Cannot read property 'id' of undefined
```

## Recent Evolution Events (sanitized)
| # | Intent | Gene | Outcome | Reason |
|---|--------|------|---------|--------|

---
*This issue was automatically created by evolver v1.29.8*
```

### 2.6 HubReview (Hub 资产评价) - ❌ 缺失

**evolver 实现**: `src/gep/hubReview.js`

**功能描述**:
- 在 solidify 后自动提交 Hub 资产评价
- 基于进化结果派生评分 (成功 4-5 分，失败 1-2 分)
- 防重复评价机制
- 记录评价历史

**评分规则**:
- 成功 + score >= 0.85: 5 分
- 成功 + score < 0.85: 4 分
- 失败 + 约束违规: 1 分
- 失败 + 无约束违规: 2 分

**API 调用**:
```
POST /a2a/assets/:assetId/reviews
{
  "sender_id": "node_xxx",
  "rating": 5,
  "content": "Outcome: success (score: 0.92)\nReuse mode: reference\n..."
}
```

### 2.8 Commitment Tracking (任务承诺期限) - ❌ 缺失

**evolver 实现**: `src/gep/taskReceiver.js` (estimateCommitmentDeadline)

**功能描述**:
- 基于任务难度估算承诺期限
- 难度 0-0.3: 15 分钟
- 难度 0.3-0.5: 30 分钟
- 难度 0.5-0.7: 60 分钟
- 难度 0.7-1.0: 120 分钟
- 尊重任务 expires_at 上限

**实现**:
```javascript
function estimateCommitmentDeadline(task) {
  const difficulty = task.complexity_score ?? localDifficultyEstimate(task);
  // 根据难度映射到期限...
  return deadline.toISOString();
}
```
