## 二、架构设计方面

### 2.1 ✅ 优势

1. **GEP 协议合规性**：完整实现了 5 个必需的 GEP 对象
   - Mutation（触发器）
   - PersonalityState（心态）
   - EvolutionEvent（记录）
   - Gene（知识模板）
   - Capsule（结果快照）

2. **模块化设计**：核心功能分离为独立类
   - `SignalExtractor`：信号检测与提取
   - `GeneSelector`：基因/胶囊选择
   - `PromptBuilder`：GEP 提示构建
   - `SolidifyEngine`：结果固化与验证

3. **MCP 协议实现**：通过 stdio 实现了 JSON-RPC 2.0 协议

4. **SQLite 存储**：使用 WAL 模式和 mmap 优化，支持自动迁移

### 2.2 ⚠️ 需要改进的地方

#### 2.2.1 Schema 版本不一致（高优先级）

```php
// src/ContentHash.php
public const SCHEMA_VERSION = '1.6.0';

// src/PromptBuilder.php  
private const SCHEMA_VERSION = '1.5.0';

// src/Database.php
private const SCHEMA_VERSION = '1.6.0';
```

**问题**：版本不一致可能导致协议混淆和迁移问题。

**建议**：统一所有 SCHEMA_VERSION 常量，创建单一的版本源。

#### 2.2.2 类职责分离不够彻底（中优先级）

`McpServer` 类超过 1000 行，同时承担了：
- MCP 协议处理
- 12 个工具的调度逻辑
- 7 个资源的管理
- 错误处理和响应格式化

**建议**：提取工具处理逻辑到单独的 `ToolHandler` 类或按功能分组。

#### 2.2.3 事件系统缺失（中优先级）

当前组件间耦合度较高，缺乏统一的内部事件/消息总线。

**建议**：引入轻量级事件总线实现组件解耦。

## 三、代码质量方面

### 3.1 ✅ 优势

- **严格类型**：所有文件使用 `declare(strict_types=1)`
- **命名规范**：遵循 PSR-4 自动加载，命名空间清晰
- **文档注释**：复杂逻辑有适当文档说明

### 3.2 ⚠️ 需要改进的地方

#### 3.2.1 类型声明不完整（中优先级）

```php
// SignalExtractor.php:275
'suppressedSignals' => new class($suppressedSignalsArray) {
    // 匿名类实现
}
```

**建议**：将匿名类提取为正式类 `SuppressedSignalSet`。

#### 3.2.2 魔法数字和硬编码值（中优先级）

阈值和常量分散在各处：
- `SignalExtractor.php`：REPAIR_LOOP_THRESHOLD = 3
- `SolidifyEngine.php`：MAX_FILES_HARD_LIMIT = 60
- 多处重复定义的约束值

**建议**：创建统一的 `ThresholdConfig` 或 `EvolverConfig` 类管理所有阈值。

#### 3.2.3 错误处理不一致（高优先级）

```php
// 方式一：返回数组
return ['ok' => false, 'error' => '...'];

// 方式二：抛出异常
throw new \RuntimeException('...');
```

**建议**：统一错误处理模式，引入专门的异常层级。

#### 3.2.4 重复代码（中优先级）

`GepAssetStore::migrateSchema()` 和 `Database::runMigrations()` 存在重复的迁移逻辑。

**建议**：将迁移逻辑集中到 `Database` 类。

## 四、安全模型方面

### 4.1 ✅ 优势

1. **源码保护**：`SourceProtector` 防止自修改核心文件
2. **安全模式**：三种模式（never/review/always）通过环境变量控制
3. **命令白名单**：验证命令只允许安全命令
4. **Shell 操作符拦截**：阻止危险操作符

### 4.2 ⚠️ 需要改进的地方

#### 4.2.1 命令白名单不够灵活（低优先级）

```php
private const ALLOWED_COMMAND_PREFIXES = ['php', 'composer', 'phpunit', 'phpcs', 'phpstan'];
```

**建议**：支持通过配置文件扩展白名单。

#### 4.2.2 输入验证不足（高优先级）

MCP 工具参数缺乏严格的输入验证，可能导致：
- 路径遍历攻击
- 超大输入导致的内存问题
- 类型不匹配错误

**建议**：为所有工具参数添加验证层。

#### 4.2.3 数据库注入风险（高优先级）

```php
// GepAssetStore.php:335
$this->db->exec("ALTER TABLE {$table} ADD COLUMN {$column} {$type}");
```

**建议**：所有动态 SQL 使用参数化查询或严格的输入净化。
